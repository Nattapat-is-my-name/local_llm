01 — Foundation

LLM คืออะไร?

Large Language Model (LLM) คือ Neural Network ขนาดใหญ่ที่ถูก Train มาเพื่อ "ทำนายข้อความถัดไป" โดยมีพื้นฐานจาก Transformer Architecture

🧠 ทำงานอย่างไร?

รับ Input (Tokens) → ผ่าน Layers ของ Attention + Feed-Forward → Output คือ Probability distribution ของ Token ถัดไป → เลือก Token ที่ดีที่สุด

📐 Transformer Architecture

ประกอบด้วย Embedding Layer + หลาย Transformer Blocks (Attention Heads + FFN) + Output Layer โดยแต่ละ Layer เรียนรู้ Feature ที่แตกต่างกัน

⚡ Why "Local"?

Run บน Hardware ของตัวเอง ไม่ต้องส่งข้อมูลออก Cloud → Privacy, Offline use, No API cost, Full control, Custom fine-tuning

Two Phases of Inference

Prefill Phase

Fast, Parallel

รับ prompt ทั้งหมด → ประมวลผลพร้อมกัน (parallel) สร้าง KV cache สำหรับทุก position ใน prompt

Output: "Time to First Token" (TTFT)

Decode Phase

Slow, Sequential

สร้างทีละ token (autoregressive) แต่ละ token ต้อง attention กับทุก token ก่อนหน้า

Output: "Time per Output Token" (TPOT)

Vocabulary: ศัพท์ที่ต้องรู้

คำศัพท์ความหมาย
Tokenหน่วยเล็กที่สุดของ text (~0.75 คำ หรือ ~4 ตัวอักษร)
Parametersค่าน้ำหนักที่เรียนรู้ — 7B = 7 พันล้านค่า
Context Windowจำนวน token สูงสุดที่โมเดลรับได้ (เช่น 8K, 32K, 128K)
Embeddingแปลง token → vector ขนาดเช่น 4096 มิติ
Temperatureควบคุม randomness — สูง=สุ่ม, ต่ำ=deterministic
Top-p / Nucleus Samplingเลือกจาก smallest set ที่รวม probability = p

05.5 — Quantization Deep Dive

Quantization คืออะไร?

Quantization คือกระบวนการลด Precision ของ Weights จาก FP32/FP16 ลงมาเป็น Integer ที่มี Bit น้อยลง เพื่อลด VRAM และเพิ่ม Speed โดยยอมเสีย Accuracy เพียงเล็กน้อย

⚙️ ลอง Adjust Quantization level
Q2 (เล็กมาก)Q3Q4Q5Q6Q7Q8 (ใกล้ full)
VRAM ที่ต้องใช้ (8B model)4.5 GB
Quality Score (เทียบ FP16)~92%
Speed vs FP16~2.2x
แนะนำสำหรับGeneral use

GGUF Quantization Naming Convention

# Format: Q{bits}_{method}_{size} Q4 = 4 bits per weight _K = K-quant method (ใช้ Importance weighting — ดีกว่า legacy) _M = Medium size variant _S = Small variant (เล็กกว่า _M เล็กน้อย, ช้ากว่าเล็กน้อย) _L = Large variant (ใหญ่กว่า, แม่นยำกว่า) _XS = Extra small # K-quant มีอยู่: Q2_K, Q3_K_S/M/L, Q4_K_S/M, Q5_K_S/M, Q6_K, Q8_0 # แนะนำ: Q4_K_M (default sweet spot), Q5_K_M (ถ้า VRAM พอ)

🎯 Sweet Spot Recommendation

สำหรับ Local use: เริ่มที่ Q4_K_M — ใช้ VRAM น้อย ยังคง Quality ดี และเป็น Default ที่ LM Studio แนะนำ ถ้า VRAM เหลือพอ ให้ลอง Q5_K_M สำหรับ Quality ที่ดีขึ้น

08 — Hardware Selection

เลือก Hardware อย่างไร?

แบ่งตาม Budget และ Use Case เพื่อให้เลือกได้ตรงจุด

🔰 STARTER

฿15,000–30,000
  • GPU: RTX 3060 (12GB) หรือ RTX 3070 (8GB)
  • RAM: 32 GB DDR4/5
  • รัน: 7B models ได้สบาย
  • Speed: ~25-40 t/s (Q4)

⚡ MID-RANGE

฿30,000–60,000
  • GPU: RTX 4070 Ti (12GB) / RTX 3090 (24GB)
  • RAM: 64 GB DDR5
  • รัน: ถึง 13-14B ได้ดี
  • Speed: ~60-90 t/s (Q4 7B)

🚀 HIGH-END

฿60,000–120,000
  • GPU: RTX 4090 (24GB) หรือ 2x RTX 3090
  • RAM: 128 GB DDR5
  • รัน: 34B ใน VRAM, 70B hybrid
  • Speed: ~110+ t/s (Q4 7B)

💎 ULTRA (Apple)

฿100,000–220,000
  • Mac Studio / Mac Pro M4 Max/Ultra
  • 192 GB Unified Memory
  • รัน: 70B+ model ใน RAM ทั้งหมด
  • ข้อดี: กินไฟน้อย, เงียบ, Portable

Key Factors ในการเลือก

1

VRAM / Unified Memory คือสิ่งสำคัญที่สุด

Model weight ต้อง fit ใน VRAM ถึงจะ run ได้เร็ว ถ้า overflow ไป RAM จะช้าลงอย่างมาก (10x+ slower) กฎง่ายๆ: VRAM ≥ Model size × 1.2

2

Memory Bandwidth สำคัญกว่า FLOPS

LLM inference เป็น Memory-bound operation ไม่ใช่ Compute-bound → GPU ที่มี Bandwidth สูง (RTX 4090: 1008 GB/s) เร็วกว่า GPU ที่มี TFLOPS สูงแต่ Bandwidth ต่ำ

3

พิจารณา Use Case

แค่ chat → 7-8B Q4 บน RTX 3060 พอ / Coding assist → 14B+ / Research/Analysis → 70B+ / Real-time response critical → เน้น GPU CUDA / Privacy + large model → Apple M-series

09 — LM Studio Guide

Host LLM ด้วย LM Studio

LM Studio คือ Desktop App สำหรับ Download, Run, และ Serve LLM บนเครื่องตัวเอง รองรับ Windows, macOS, Linux มี GUI ที่ใช้งานง่าย และ Built-in OpenAI-compatible API server

⬇️

1. Download & Install LM Studio

ไปที่ lmstudio.ai → Download ตาม OS → ติดตั้งปกติ

🔍

2. ค้นหา & Download Model

กดที่ 🔍 Search tab → พิมพ์ชื่อ model เช่น "llama 3.1 8b instruct" → เลือก Quantization ที่เหมาะกับ VRAM ของคุณ → กด Download

💡 Tips

LM Studio แสดง Estimated VRAM ให้เห็น ถ้า chip สีเขียว = VRAM พอ / สีเหลือง = จะ offload บางส่วน / สีแดง = ไม่พอเด็ดขาด

💬

3. Chat Interface

กด Chat icon → New Chat → เลือก model ที่ Download มา → ตั้งค่า System Prompt (ถ้าต้องการ) → เริ่ม Chat ได้เลย

⚙️

4. ปรับ Model Settings ที่สำคัญ

Parameterความหมายแนะนำ

GPU Layersกี่ layers ที่ offload ไป GPU-1 = ทั้งหมด (ถ้า VRAM พอ)
🌐

5. เปิด Local API Server (OpenAI-compatible)

กด "Local Server" icon → เลือก model → กด "Start Server" → Server จะ listen ที่ http://localhost:1234

Python — FastAPI Backend

from fastapi import FastAPI, HTTPException from pydantic import BaseModel from typing import Optional, List import openai app = FastAPI() openai.api_base = "http://localhost:1234/v1" openai.api_key = "no-key" conversations = {} class ChatRequest(BaseModel): user_id: str message: str system_prompt: Optional[str] = None temperature: float = 0.7 @app.post("/chat") async def chat(req: ChatRequest): try: if req.user_id not in conversations: conversations[req.user_id] = [] history = conversations[req.user_id] messages = [] if req.system_prompt: messages.append({"role": "system", "content": req.system_prompt}) messages.extend(history) messages.append({"role": "user", "content": req.message}) response = openai.ChatCompletion.create( model="local-model", messages=messages, temperature=req.temperature ) reply = response.choices[0].message.content history.append({"role": "user", "content": req.message}) history.append({"role": "assistant", "content": reply}) return {"reply": reply, "conversation_id": req.user_id} except Exception as e: raise HTTPException(status_code=500, detail=str(e))
